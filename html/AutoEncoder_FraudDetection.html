Dataset - Credit Card Fraud Detection¶

Link:
https://www.kaggle.com/mlg-ulb/creditcardfraud

Contexto:
de extrema importncia que as empresas identifiquem e reconheam transaes fraudulentas com carto de crdito, para que os clientes no sejam cobrados pelos itens que no compraram e as empresas no tomem prejuzo devido fraude.

Contedo:
O dataset contm transaes realizadas com cartes de crdito em setembro de 2013 por portadores de cartes europeus. Este conjunto de dados apresenta transaes que ocorreram em dois dias, nas quais temos 492 fraudes em 284.807 transaes. O conjunto de dados altamente desequilibrado, a classe positiva (fraudes) representa 0,172% de todas as transaes.

O dataset contm apenas variveis de entrada numricas que so o resultado de uma transformao PCA. Infelizmente, devido a problemas de confidencialidade, no possvel utilizar os recursos originais e mais informaes bsicas sobre os dados. Features de V1, V2, at V28 so os principais componentes obtidos com o PCA, os nicos recursos que no foram transformados com o PCA so 'Tempo' e 'Valor'. O recurso 'Tempo' contm os segundos decorridos entre cada transao e a primeira transao no conjunto de dados. O recurso 'Valor' o valor da transao, esse recurso pode ser usado para aprendizado sensvel a custos dependente de exemplo. O recurso 'Classe' a varivel de resposta e assume o valor 1 em caso de fraude e 0 em caso contrrio.

Resoluo:

Para resoluo do problema, utilizamos Autoendoder, que uma tcnica de aprendizado no supervisionado de Deep Learning, para realizar reduo de dimensionalidade e identificar valores extremos.

A framework Keras foi o escolhido para usar na implementao do modelo de rede neural utilizado pelo Autoencoder.

5. Autoencoder¶

5.1. Introdução¶

Para o treinamento do Autoencoder so utilizados apenas os dados referentes s transaes no fraudulentas. Durante o treinamento, o codificador ver milhes de transaes no fraudulentas com carto de crdito.

Dessa forma, o Autoencoder codifica as informaes mais relevantes dos dados no fraudulentos. Intuitivamente, podemos dizer que dessa maneira o algoritimo aprende apenas o que uma transao de carto de crdito absolutamente normal.

E esse conceito aprendido de uma transao normal com carto de crdito pode ser encontrado como a representao latente na camada intermediria, usada para recriar os recursos originais usados como dados de entrada para o Autoencoder.

Aps o treinamento com os dados no fraudulentos, podemos finalmente usar o Auto Encoder para detectar as anomalias (fraudes).

Com isso, possvel mostrar para o Auto Encoder os dados de ambos os tipos - anomalias e dados normais. Como antes, os recursos de entrada so codificados pelo Autoencoder em uma representao latente que usada para reconstruir a entrada. Os dados de fraude vo possuir um erro de reconstruo muito superior aos dados normais, pois o Auto Encoder aprendeu a codificar dados de transaes normais.

OBS:

Caso o Autoencoder seja treinado corretamente, esperamos que o erro entre a sada e a entrada seja muito pequeno para dados normais.

Entretanto, esse no o caso das anomalias ou, no nosso caso, transaes fraudulentas de dados.

Durante o treinamento o codificador automtico aprendeu apenas os conceitos e caractersticas da instncia normal de dados. Isso significa que os pesos do Autoencoder foram ajustados apenas para codificar e reconstruir dados normais (transaes no fraudulentas).

Se tentarmos codificar uma transao fraudulenta, a representao latente dessa transao seria significativamente diferente da representao latente de uma transao normal.

Como resultado direto, a entrada reconstruda diferiria ainda mais da entrada original, resultando em um erro muito maior do que no caso de transaes normais.

O conhecimento do fato de que dados de entrada fraudulentos resultam em valores de perda mais altos da funo de perda de erro quadrtico mdio (MSE - Mean Squared Error) pode ser usado ao nosso favor. Tudo o que precisamos fazer encontrar um limite de perda que distinga os dados normais dos fraudulentos (threshold).

Na prtica, isso significa que instncias de dados para as quais obtemos um valor de perda acima desse limite classificam essa instncia como uma transao fraudulenta.

Por outro lado, instncias de dados com valores de perda abaixo desse limite podem ser consideradas dados normais ou transaes no fraudulentas.

Caso 1: O valor da perda MSE para um recurso de entrada est acima do limiar de perda  o recurso de entrada  uma transao fraudulenta
Caso 2: O valor da perda MSE para um recurso de entrada est abaixo do limite  o recurso de entrada  uma transao no fraudulenta

encoder = Dense(14, activation="relu", activity_regularizer=regularizers.l2(10e-5))(encoder)

# Camadas de decoding

decoder = Dense(18, activation='relu')(decoder)
decoder = Dense(input_dim, activation='relu')(decoder)

autoencoder = Model(inputs=input_layer, outputs=decoder)

autoencoder.compile(optimizer='adam', loss='mean_squared_error', metrics=['accuracy'])

7. Métricas¶

7.1 Precisão vs Revocação (Recall)¶

Preciso e Recall so definidos das seguintes maneiras:

$$\text{Preciso} = \frac{\text{verdadeiros positivos}}{\text{verdadeiros positivos} + \text{falsos positivos}}$$$$\text{Revocao} = \frac{\text{verdadeiros positivos}}{\text{verdadeiros positivos} + \text{falsos negativos}}$$

Preciso (tambm chamada de valor preditivo positivo) a frao de instncias recuperadas que so relevantes.

Revocao (tambm conhecida como sensibilidade) a frao de instncias relevantes que so recuperadas.

Tanto preciso quanto revocao so, portanto, bases para o estudo e compreenso da medida de relevncia.

Alto recall, mas baixa preciso significa muitos resultados, a maioria dos quais tem pouca ou nenhuma relevncia. Quando a preciso alta, mas o recall baixo, temos o oposto - poucos resultados retornados com uma relevncia muito alta. Normalmente, o resultado ideal seria ter uma alta preciso e alto recall.

7.2 F1 Score¶

F1 Score uma medida que combina preciso e revocao a mdia harmnica entre preciso e revocao

$${\displaystyle F1=2\cdot {\frac {\mathrm {preciso} \cdot \mathrm {revocao} }{\mathrm {preciso} +\mathrm {revocao} }}}$$

Esta medida aproximadamente a mdia de ambas quando seus valores esto prximos, e de maneira mais geral, o quadrado da mdia geomtrica dividido pela mdia aritmtica. H vrias razes pelas quais o F1 Score pode ser criticado em casos especficos devido ao seu vis como mtrica de avaliao.

Fonte: https://pt.wikipedia.org/wiki/Precis%C3%A3o_e_revoca%C3%A7%C3%A3o
